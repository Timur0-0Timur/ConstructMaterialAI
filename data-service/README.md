# 🏭 Pump ETL & ML Data Pipeline

## 📌 Описание проекта
Данный проект представляет собой автоматизированный ETL-конвейер (Extract, Transform, Load) для обработки сырых инженерных данных промышленных насосов. 

**Главная бизнес-цель:** Подготовка качественного датасета для обучения ML-модели, которая будет **предсказывать физический вес насосного оборудования** на основе его эксплуатационных характеристик (подача, напор, мощность и т.д.). 

Проект поддерживает два режима:
1. **Batch Processing:** Массовая обработка Excel-файлов для формирования обучающей выборки.
2. **Real-time API:** Обработка одиночных запросов от приложения/пользователя с мгновенным расчетом признаков.

## 🚀 Ключевые особенности
* **REST API на FastAPI:** Готовый эндпоинт для интеграции с другими сервисами или фронтендом.
* **Продвинутая очистка (Vectorized Cleaning):** Обработка "грязных" строк из Excel (разные разделители, спецсимволы) через Regex.
* **Data Enrichment:** Автоматическое восстановление пропущенных параметров (RPM, мощность) в новых данных на основе эталонного ML-датасета.
* **Feature Engineering:** Генерация физически обоснованных признаков (`useful_kw`, `specific_speed`) с высокой корреляцией к весу.
* **Умное кэширование:** Конвертация тяжелых `.xlsx` в `.csv` для ускорения I/O при повторных запусках.

## 🧠 Аналитика и Feature Engineering
* **Трансформация веса:** Используется логарифмирование (`weight_log`) для борьбы со смещением распределения.
* **Полезная мощность (`useful_kw`):** Напор × Расход. Ключевой признак с корреляцией **0.84** к целевой переменной.
* **Фильтрация:** Удалены признаки с низкой значимостью (например, КПД с корреляцией 0.01).

## 🛠 Установка и запуск

### 1. Подготовка окружения
```bash
git clone <URL_репозитория>
cd data-service
python -m venv .venv
source .venv/bin/activate # или .venv\Scripts\activate для Windows
pip install -r requirements.txt
```

### 2. Запуск через API (Режим сервиса)
Для запуска локального сервера используйте `uvicorn`:
```bash
uvicorn api.api:app --reload
```
После запуска документация доступна по адресу: **[http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)**

**Пример запроса (JSON):**
```json
{
  "tag": "P-101",
  "flow_rate": 15.5,
  "fluid_head": 120.0,
  "rpm": null,
  "spec_gravity": null
}
```
*Если поля `rpm` или `spec_gravity` не заполнены, сервис восстановит их через Enricher.*

### 3. Пакетная обработка (Batch)
Для генерации тренировочного датасета из всех файлов в папке `data`:
```bash
python pipelines/etl_pipeline.py
```
