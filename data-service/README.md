# 🏭 Pump ETL & ML Data Pipeline

## 📌 Описание проекта
Данный проект представляет собой автоматизированный ETL-конвейер (Extract, Transform, Load) для обработки сырых инженерных данных промышленных насосов. 

**Главная бизнес-цель:** Подготовка качественного датасета для обучения ML-модели, которая будет **предсказывать физический вес насосного оборудования** на основе его эксплуа যুক্তরাজных характеристик (подача, напор, мощность и т.д.). Также реализован Inference-пайплайн для обработки реальных (новых) данных с автоматическим обогащением (enrichment) пропущенных значений.

## 🚀 Ключевые особенности
* **Продвинутая очистка (Vectorized Cleaning):** Обработка грязных строковых данных из Excel (разные форматы разделителей тысяч и дробей) с помощью векторизованных регулярных выражений (Regex).
* **Умное кэширование:** Конвертация "тяжелых" `.xlsx` файлов в скрытые `.csv` при первом чтении для многократного ускорения последующих запусков пайплайна.
* **Feature Engineering:** Генерация новых физически обоснованных признаков на основе предметной области.
* **Data Enrichment:** Восстановление пропущенных параметров в новых данных на основе эталонного ML-датасета (поиск по напору и расходу).
* **Облачная интеграция:** Автоматическая выгрузка очищенных витрин данных в PostgreSQL (Neon DB) через SQLAlchemy.

## 🧠 Feature Engineering и Аналитика (EDA)
В процессе исследовательского анализа данных (EDA) и создания признаков были сделаны следующие оптимизации:

1. **Трансформация целевой переменной:** Исходный вес насосов имеет сильное смещение (Right Skew). Для стабильного обучения ML-моделей применяется логарифмирование целевой переменной (`weight_log`), что приводит её к нормальному распределению.
2. **Удаление шума:** Колонка `pump_eff` (КПД) была исключена из итогового датасета, так как её корреляция с целевой переменной составила всего **0.01**.
3. **Генерация новых признаков:**
   * `useful_kw` (Полезная мощность = Напор × Расход). **Корреляция с весом составила 0.84!** Это ключевой предиктор для модели.
   * `specific_speed` (Удельная быстроходность) и `diameter_proxy` (Оценка диаметра колеса) — добавлены физические параметры, дающие прирост корреляции (~0.22 - 0.23).

## 🛠 Установка и запуск

### 1. Подготовка окружения
Клонируйте репозиторий и создайте виртуальное окружение:
```bash
git clone <URL_вашего_репозитория>
cd <папка_проекта>/data-service
python -m venv .venv
source .venv/bin/activate  # Для Mac/Linux
# .venv\Scripts\activate   # Для Windows
pip install -r requirements.txt
```

### 2. Настройка базы данных
Создайте файл `.env` в корневой папке (`data-service/`) и добавьте доступы к PostgreSQL (Neon):
```env
DB_HOST=ваш_хост
DB_PORT=5432
DB_NAME=имя_базы
DB_USER=пользователь
DB_PASSWORD=пароль
```

### 3. Запуск пайплайнов
Для генерации тренировочного датасета (`dataset_ml.csv`) и отправки его в БД:
```bash
python pipelines/etl_pipeline.py
```

Для обработки новых "сырых" данных с использованием обогащения (`dataset_inference.csv`):
```bash
python pipelines/inference_pipeline.py
```

### 4. Визуализация и отчеты
Для генерации графиков (корреляции, распределения, scatter plots) запустите скрипты из папки analysis:
```bash
python analysis/visualize.py
python analysis/eda_visualizer.py
```
*(Сгенерированные `.png` файлы сохранятся в корневую директорию).*
